1. 본 프로젝트는 상업적 목적이 아닌, 1인 개발자의 엣지(Edge) 인공지능 기술 실증(PoC) 및 개인 연구를 목적으로 진행되었습니다.
2. 따라서 상용 수준의 장기 내구성 테스트나 완벽한 교차 검증보다는, 자원이 극도로 제한된 환경에서 모델이 실제 구동되는 것을 확인하는 데 초점을 맞추었습니다.
3. 시스템 아키텍처 설계 및 코드 작성의 90% 이상 인공지능(Google Gemini Pro)과의 협업을 통해 완성되었습니다.
4. LCD 패널 구동의 경우, 기기를 완성하여 성공적으로 실험을 마친 후 해체하고 나서야 사진/영상 기록을 남기지 않았음을 깨달았습니다. 하지만 시스템의 핵심 관제 로직과 특정 안면 패턴에 반응하는 보안 종료 시퀀스(이스터에그)는 완벽하게 동작함을 확인했습니다.(기기 완성 사진만 3장 있는거 나노바나나로 편집했습니다.)
5. 본 프로젝트는 Apache 2.0 라이선스를 따르며 Zenodo에 등록되었으므로, 해당 규정 범위 내에서 자유롭게 활용 및 변형이 가능합니다.
6. 계속 강조하지만, 원활한 구동을 위해서는 반드시 라즈베리파이 3의 메모리 최적화가 필수적이며, 기기 쿨링 상태에 따른 CPU 오버클럭을 강력히 권장합니다. (오버클럭 적용 시, 외부 전원이 연결된 쿨링팬 부착이 필수입니다.)

8. # ⚙️ Raspberry Pi 3 기반 Edge AI 하드웨어 성능 최적화

본 문서는 제한된 연산 자원(1GB RAM)을 가진 Raspberry Pi 3 Model B 보드에서 딥러닝 기반 운전자 상태 모니터링(DSM) 시스템을 안정적으로 구동하기 위한 하드웨어 수준의 성능 튜닝(Performance Tuning) 및 메모리 최적화 가이드라인입니다.

## 1. 최적화 목표 (Objectives)

본 튜닝의 목적은 자원 제약 환경(Resource-Constrained Environment)에서 다음의 성능 지표를 달성하는 것입니다.

1.  **CPU 클럭 최적화:** AI 추론 지연시간(Latency) 최소화를 위한 안정적인 오버클럭(Overclocking).
2.  **메모리 효율 극대화:** ZRAM(가상 압축 메모리) 도입을 통한 OOM(Out of Memory) 현상 원천 차단.
3.  **시스템 자동화:** 전원 인가 시 사용자 개입 없이 구동되는 완전 자립형(Standalone) 데몬 구축.

---

## 2. CPU 오버클럭 (Overclocking) 설정

운영체제의 부트 로더 설정을 수정하여 기본 클럭(1.2GHz)을 상향 조정합니다. (방열판 및 5V 3A 이상의 안정적인 외부 전원 공급이 필수적으로 요구됩니다.)

### 2.1. /boot/firmware/config.txt 수정
```bash
sudo nano /boot/firmware/config.txt
```
파일 하단에 다음 파라미터를 추가합니다. (`force_turbo` 옵션은 발열에 의한 스로틀링을 유발하므로 배제합니다.)
```ini
[pi3]
# CPU 코어 주파수를 1.35GHz로 상향 조정
arm_freq=1350
# GPU 및 L2 캐시 주파수 조정
core_freq=500
# SDRAM 주파수 상향 (메모리 대역폭 확장)
sdram_freq=500
# 오버클럭에 따른 코어 전압 상향 조정 (안정성 확보)
over_voltage=4
```

### 2.2. 상태 모니터링 명령어
오버클럭 적용 후, 다음 명령어를 통해 현재 클럭 및 스로틀링(Throttling) 발생 여부를 확인합니다.
*   클럭 속도 확인: `vcgencmd measure_clock arm` (1350000000 내외 정상)
*   스로틀링 확인: `vcgencmd get_throttled` (`0x0` 출력 시 정상)

## 3. 메모리 최적화 및 ZRAM 구성

물리적 RAM(1GB)의 한계를 극복하기 위해 디스크 스왑(Disk Swap) 대신 속도가 압도적으로 빠른 메모리 내 압축(ZRAM) 기술을 적용합니다.

### 3.1. ZRAM 초기화 스크립트 작성
```bash
nano /home/pi/zram_init.sh
```
다음 내용을 작성합니다. (가장 압축/해제 속도가 빠른 `lz4` 알고리즘을 채택합니다.)
```bash
#!/bin/bash
# ZRAM 커널 모듈 로드
modprobe zram
sleep 2

# lz4 알고리즘 지정 및 1GB 가상 용량 할당
echo lz4 > /sys/block/zram0/comp_algorithm
echo 1G > /sys/block/zram0/disksize

# Swap 포맷 및 활성화 (우선순위를 가장 높게 설정)
mkswap /dev/zram0
swapon -p 100 /dev/zram0
```
스크립트 실행 권한 부여: `chmod +x /home/pi/zram_init.sh`

### 3.2. Systemd 서비스 등록 (자동화)
Legacy 방식인 `rc.local` 대신, 시스템 기동 시 최우선으로 메모리를 확보하기 위해 `systemd` 서비스로 등록합니다.
```bash
sudo nano /etc/systemd/system/zram-init.service
```

```ini
[Unit]
Description=ZRAM Virtual Memory Initialization
After=local-fs.target

[Service]
Type=oneshot
ExecStart=/home/pi/zram_init.sh
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target
```
서비스 활성화 및 적용:
```bash
sudo systemctl daemon-reload
sudo systemctl enable zram-init.service
```

## 4. 커널 파라미터 (sysctl) 튜닝

ZRAM의 효율을 극대화하기 위해 리눅스 커널의 가상 메모리 관리 정책을 수정합니다. ZRAM은 속도가 빠르므로, 메모리가 완전히 소진되기 전에 선제적으로 데이터를 압축하도록 `swappiness` 값을 상향해야 합니다.
```bash
sudo nano /etc/sysctl.conf
```
파일 하단에 다음 값을 추가합니다.
```ini
# ZRAM 활용을 적극적으로 유도하여 OOM 방지 (기본 60 -> 80 상향)
vm.swappiness=80
# VFS 캐시 유지력을 높여 파일 시스템 I/O 병목 완화
vm.vfs_cache_pressure=50
# 디스크 쓰기 대기 데이터를 메모리에 일정 부분 유지
vm.dirty_ratio=10
vm.dirty_background_ratio=5
```
변경 사항 즉시 적용: `sudo sysctl -p`

4.  **[Inference]** AI 연산 중 발생하는 메모리 스파이크를 ZRAM이 흡수하여 시스템 멈춤(Freeze) 방지
